## 1. Purpose

The Training & Enablement Guide defines the standards, learning paths, responsibilities, and documentation required to ensure staff, contractors, and partners are properly trained in the tools, systems, and operational disciplines used across Moko Consulting.

The guide supports both onboarding and continuous education.

## 2. Training Structure

Training programs must include:

* Learning objectives
* Required reading
* Hands-on exercises
* Assessments (when applicable)
* Documentation of completion

Training materials stored under:

```
docs/training/materials/
```

## 3. Required Training Areas

### 3.1 Security

* Password hygiene
* MFA usage
* Incident reporting
* Data handling requirements

### 3.2 Operations

* Ticketing workflows
* Maintenance windows
* Change approval expectations

### 3.3 Engineering

* Coding standards (MokoStandards)
* Git workflows
* Testing practices

### 3.4 Compliance

* Evidence documentation
* Audit expectations

### 3.5 Tools & Systems

* Dolibarr modules
* Internal admin tools
* Deployment automation

## 4. Onboarding Path

Every new team member must:

1. Complete the Onboarding Guide
2. Receive system access based on Access Management Guide
3. Complete mandatory training modules
4. Sign required acknowledgements
5. Perform hands-on onboarding tasks

## 5. Continuing Education

Teams must complete:

* Annual security refresher
* Tool updates training
* Process updates training
* Role-specific training as technologies evolve

Training events logged at:

```
docs/training/logs/
```

## 6. Assessment & Certification

For critical systems, training may include:

* Knowledge checks
* Scenario-based assessments
* Hands-on evaluations

Certificates stored under:

```
docs/training/certificates/
```
