# MokoCRM Glossary

## Overview

This glossary defines terminology specific to MokoCRM, our Dolibarr-based CRM/ERP platform.

## A-C

**Action** - An activity or event (call, meeting, email, task) logged in MokoCRM
**API Token** - Authentication credential for MokoCRM REST API access
**Agenda** - Calendar and scheduling module displaying all activities

**Backlog** - Collection of identified but unscheduled project tasks
**Bank Account** - Configured financial account for transaction tracking

**Company (Third Party)** - Organization or business entity (customer, prospect, supplier, partner)
**Contact** - Individual person associated with a company
**Contract** - Formal agreement tracking terms, services, and renewal dates
**Custom Module** - Dolibarr extension developed for MokoCRM

## D-F

**Dashboard** - Main interface with widgets and statistics (Moko-enhanced)
**Dictionary** - System-wide dropdown lists ensuring data consistency
**Dolibarr** - Open-source ERP/CRM foundation for MokoCRM
**Document Template** - Pre-formatted, customizable business document

**Entity** - Separate business unit in multi-company mode
**ERP** - Enterprise Resource Planning software
**Event** - Scheduled occurrence in agenda

**Feature Set** - Licensing option determining available functionality
**Fiscal Year** - 12-month accounting period

## G-I

**Global Administrator** - User with unrestricted system access
**Group** - Collection of users with shared permissions

**Hook** - Extension point for custom module functionality
**htdocs** - Root directory of MokoCRM installation

**Intervention** - Service call, maintenance visit, or support ticket
**Invoice** - Billing document requesting payment

## J-L

**Job Position** - Role or title within an organization

**KPI** - Key Performance Indicator for business objectives
**Knowledge Base** - Repository of articles and solutions

**Lead** - Potential customer showing interest
**License** - GPL-3.0 usage rights with Moko branding

## M-O

**Margin** - Difference between cost and selling price
**Member** - Individual/organization in membership module
**Migration Script** - SQL code for database updates
**MokoCRM** - Moko Consulting branded CRM/ERP based on Dolibarr
**Module** - Functional component that can be enabled/disabled
**Module Descriptor** - PHP class defining module properties (modMyModule.class.php)
**Module Number** - Unique identifier (100000-999999) for custom modules

**Notification** - Automated alert for specific events

**Object** - Business entity (invoice, contact, product) in OOP architecture
**Opportunity** - Potential sale tracked with stages and probabilities
**Order** - Confirmed customer request for products/services

## P-R

**Permission** - Access grant to specific functionality
**Product** - Item offered for sale or tracked in inventory
**Project** - Defined work with tasks, timelines, and resources
**Proposal (Quote)** - Formal offer to customer

**Queue** - List of items awaiting processing

**RACI Matrix** - Responsibility framework (Responsible, Accountable, Consulted, Informed)
**Reference** - Unique identifier for MokoCRM objects
**Reminder** - Automated notification for future action
**RestrictedArea** - Security function checking user permissions
**RFC** - Request for Change

## S-T

**Service** - Intangible product (consulting, support, subscription)
**Shipment** - Delivery or receipt of products
**Sprint** - Fixed time period (usually 2 weeks) for Agile work completion
**Stakeholder** - Anyone with interest in a project
**Stock** - Product quantities in warehouses
**Sudo Mode** - Elevated privilege state for sensitive operations

**Tag** - Label for categorization and filtering
**Task** - Unit of work within a project
**Template** - Pre-configured setup for documents, projects, or workflows
**Third Party** - Dolibarr term for companies/organizations
**Trigger** - Event hook firing when specific actions occur
**Timesheet** - Record of time spent on projects

## U-W

**Update Server** - Web service providing update notifications
**User** - Individual with login credentials and permissions
**User Group** - Collection of users sharing permissions

**VAT** - Value Added Tax on products and services
**Version** - Release number (semantic versioning: major.minor.patch)
**View** - Saved filter or display configuration

**Warehouse** - Storage location for inventory
**Webhook** - HTTP callback sending real-time data to external systems
**Widget** - Dashboard component (Moko-enhanced)
**Workflow** - Defined sequence of steps for business processes

## X-Z

**XML Manifest** - Module configuration file defining metadata

**Year-End Closing** - Accounting process finalizing fiscal year records

**Zero-Stock Alert** - Notification when product stock reaches zero
**Zone** - Geographic region for shipping, territory, or VAT rules

## MokoCRM-Specific Terms

**Moko Module** - Custom Dolibarr module by Moko Consulting
**MokoCRM Instance** - Single MokoCRM installation
**Moko Support Portal** - Dedicated support at support.mokoconsulting.tech
**Moko Theme** - Customized branding and visual appearance
**Moko Workflow Preset** - Pre-configured business processes
