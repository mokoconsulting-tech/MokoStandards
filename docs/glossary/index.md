# MokoStandards Glossary

## Purpose

This glossary provides canonical definitions for terms, acronyms, and concepts used throughout MokoStandards documentation. It ensures consistent terminology across policies, guides, checklists, and technical documentation.

## Usage

- **Bold terms** in documentation should link to glossary definitions
- Use exact glossary terminology in formal documents
- Propose new terms via pull request
- Request clarification via issues

### Core Concepts

**MokoStandards**
: The authoritative coding standards and documentation framework maintained by Moko Consulting

**MokoCRM**
: Moko Consulting's CRM platform based on Dolibarr ERP/CRM

**MokoWaaS**
: Moko Consulting's Website as a Service platform based on Joomla CMS

**Canonical Document**
: A mandatory documentation artifact defined by MokoStandards policy

**Enterprise Readiness**
: The state of having security automation, compliance controls, and operational maturity suitable for enterprise deployment

**Policy Document**
: A binding, mandatory standards document that defines requirements

**Guide Document**
: A non-binding, informative document that provides implementation guidance

**Checklist Document**
: A verification document with binary criteria for validation

**Template**
: A reusable document or code structure that demonstrates standards compliance
